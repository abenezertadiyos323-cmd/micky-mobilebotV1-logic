## Part 3 — The Live Bot Still Replies With Old Wording. Why?

There is only one explanation consistent with both facts:

> **The live n8n workflow is NOT running the code in the current file.**

The n8n Code node does not read from a file at execution time. When you edit the Code node's JS inside the n8n workflow editor and click Save (the workflow-level Save button), n8n stores the code text directly into its internal database. The Code node then runs from that stored text at execution time.

**The workflow.json file on disk is a separate export artifact.** It does NOT feed the live execution engine directly. Editing the file and saving it to disk does NOT update what n8n executes. The two are completely independent.

**What this means concretely:**

The workflow.json you are editing (and that I am reading) is a local export copy. Every change made to it — correctly — updates the file. But unless those changes are imported into the live n8n instance via the n8n UI (by re-opening the workflow editor, manually updating each code node, and clicking the global Save button), **the live bot continues to run the old code that was last saved inside n8n's database**.

The Ethiopic-address version, the missing `store_info` whitelist, the old `if (!reply_text)` guard — all of those are what n8n is currently executing from its internal database. The file is ahead of the live execution.

## Part 7 — Line-Count Truncation Risk (One Remaining Issue)

After the store_info branch fires and `reply_text` is set, Validation's line-trimmer runs unconditionally:

```js
const replyLines = reply_text.split(/\r?\n/).map(line => line.trim()).filter(Boolean);
const closingLines = [];
if (budgetFallbackNotice) closingLines.push(budgetFallbackNotice);
if (closeQuestion) closingLines.push(closeQuestion);
if (!replyLines.includes(storeCtaText)) closingLines.push(storeCtaText);
const bodyLines = replyLines.filter(line => !closingLines.includes(line));
const maxBodyLines = Math.max(0, 3 - closingLines.length);
reply_text = bodyLines.slice(0, maxBodyLines).concat(closingLines).join('\n').trim();
```

For a `store_info` turn:
- `budgetFallbackNotice = null` (no budget)
- `closeQuestion = null` (`flowIsBuy = false` for this message)
- `storeCtaText` IS in `closingLines` → `closingLines = ['For more phones and accessories, visit our store using the button below.']`
- `maxBodyLines = 3 - 1 = 2`

So the address text line is kept (as body), and the CTA is appended. 2 body lines + 1 CTA line = 3 total. This is correct and within the 3-line limit.

**No truncation issue.** The line count is fine.
